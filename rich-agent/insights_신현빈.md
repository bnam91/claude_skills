# 신현빈 특이점 & 인사이트 로그

> 대화를 통해 발견된 고유 성향, 왜곡된 판단, 비상식적 패턴을 기록한다.
> 기준: AI 권고안, 부자들의 일반적 패턴과 다른 지점 / 고유 성향이 드러나는 순간.
> 발견할 때마다 누적 업데이트.

---

## 🟠 판단 왜곡 / 맹점

### 1. 코딩 과몰입으로 돈 되는 행동을 스스로 막음
- 쇼핑몰이 잘 팔리는 상황 → 콘텐츠 더 만들면 더 팔림 → 근데 코딩에 꽂혀서 콘텐츠 못 만듦
- **왜곡**: "코딩이 생산적인 일이다"는 착각. 현빈에게 코딩은 취미에 가깝고, 콘텐츠가 돈임.
- **부자 패턴과의 차이**: 성공한 커머스 사업가는 본인이 가장 잘하는 것(콘텐츠/세일즈)에 집중하고 나머지는 위임. 현빈은 반대로 함.

### 2. 콘텐츠 대체자를 못 찾는다고 포기 (위임 어려움과 연결)
- "나 대신 콘텐츠 만들 사람 찾기 힘들어" → 찾는 노력보다 찾기 어렵다는 결론이 먼저
- **실제 원인**: 상대가 어떻게 할지 미리 보이기 때문에 믿고 맡기기가 힘든 것. "내가 하는 게 낫겠다"로 귀결됨 (검증됨).
- **맹점**: 70%짜리 대체자 + 현빈 퇴고 구조도 가능한데 100%를 기대하며 포기.
- → 위임 못 하면 현빈이 영원히 SPOF. 1000억 구조 불가능.

### 3. 실행보다 설계가 길다 (검증됨)
- 마케터+인지심리+UX 배경 → 구조를 완전히 이해하고 싶어함
- 시작 전 분석/설계에 시간이 과도하게 투입됨
- **부자 패턴과의 차이**: 성공한 사업가는 60% 이해하면 실행하고 나머지는 하면서 채움. 현빈은 이해가 완성돼야 움직이려는 경향.
- → "완성 > 완벽" 원칙을 의식적으로 강제 적용 필요.

### 4. 본인이 다 보이니까 위임이 힘듦 (검증됨)
- 타인 이해력, UX적 시각이 뛰어나서 상대가 어떻게 할지 미리 보임
- "저 사람이 하면 이렇게 되겠다" → "그냥 내가 하는 게 낫겠다"
- 콘텐츠 대체자 못 찾는 문제의 실제 뿌리이기도 함
- **왜곡**: 위임은 완벽한 대체가 아니라 레버리지. 80%짜리 위임이 현빈이 직접 하는 것보다 사업엔 이득.
- → 위임 기준을 "현빈만큼 잘하는가"가 아닌 "현빈 시간을 버는가"로 바꿔야 함.

## 🔵 고유 성향 (강점이기도 함)

### 1. 박리다매 체질이 아님 (검증됨)
- 시도했으나 돈만 날림. 본인도 인지함.
- 현빈의 방식은 **프리미엄 브랜딩 + 라포**. 이걸 억지로 바꾸려 하면 실패.
- → 사업 방향 제안 시 박리다매/대량판매 구조는 배제할 것.

### 2. 잘 되면 지루해서 다른 걸 파기 시작함
- 쇼핑몰이 자동으로 팔리자 → 코딩으로 이동
- ENTP + ADHD의 전형적 패턴. 문제가 해결되면 흥미 상실.
- **리스크**: 잘 되는 사업을 방치하다 기회 놓침.
- → 성장 중인 사업은 강제로 집중 유지하는 외부 장치 필요.

### 3. AI/자동화로 안 되는 걸 빠르게 인정함
- 콘텐츠 자동화 시도 → 라포/인간미 재현 불가 → 빠르게 포기
- 많은 사람이 AI로 될 거라 믿고 계속 시도함. 현빈은 현실적으로 판단하고 끊음.
- **강점**: 매몰비용 집착 없음. 빠른 포기 가능.

### 4. 확률적 배팅형이지만 감각으로 먼저 결론 냄
- 논리적으로 설명하나, 실제론 직감이 먼저이고 이유는 나중에 붙이는 경향 추정

## 🟣 rich-agent 운영 방식 지침

### 어설픈 넛지/프레이밍은 역효과 — 수준 높은 설계 필요 (핵심)
- 마케터+인지심리+UX 배경으로 아마추어 기법은 즉시 간파함
- 간파당한 설계는 신뢰를 떨어뜨리고 오히려 저항을 만듦
- **요구 수준**: 알면서도 걸리는 설계. 기법을 인식해도 효과가 있는 구조.
- **rich-agent 설계 원칙:**
  - 설득보다 **환경 설계** — 올바른 선택이 자연스럽게 되는 구조로 만들기
  - **사전 커밋** — 현빈이 동의한 원칙으로 나중의 판단을 미리 묶어두기
  - **마찰 설계** — 잘못된 방향(코딩 과몰입 등)엔 불편함, 올바른 방향엔 쉬움
  - **타이밍** — 현빈의 에너지/상태에 맞춰 개입 시점 선택
  - **현빈 자신의 논리로 현빈을 설득** — 외부 논리보다 본인 가치관/원칙에서 도출
  - 동기부여 포장 불필요. 구조가 행동을 만들게 설계할 것

### 1. rich-agent ↔ PM 다층 에이전트 프레임워크 (2026-03-07 구상)
- **구조**: PM들(pm-gg, pm-cc, pm-xx 등) → 각자 프로젝트만 보고 업무 요청
- **rich-agent** → 전체 조망 후 피드백 → PM들이 재검토/재요청
- **소통 채널**: 노션 기반
- **회의 사이클**: 오전/저녁/주간 회의
- **핵심 가치**: 현빈은 회의 때만 판단. 나머지는 시스템이 운영.
- **의의**: 이게 완성되면 현빈의 시간이 콘텐츠와 전략 결정에만 투입되는 구조 실현 가능.
- **현재 상태**: 뼈대 구상 완료, 애자일테이블 생성됨. 내용 채우기 + 프레임워크 구현 남음.
